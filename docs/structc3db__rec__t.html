Detailed Description

Common validated-record view.

c3db_rec_t owns no storage. It wraps a caller-provided byte buffer and exposes the logical state shared by DBF records and compact cached records: active, deleted or corrupt, plus the physical data slot that contains the last valid payload metadata. Derived classes decide whether those values are stored as object attributes or inside the wrapped bytes.

DELETE_BLOCK stores free-list linkage for deleted records. DATA_BLOCK stores the sequence, generation cycle and CRC for one payload. Derived classes define the concrete byte layout and expose the current DATA_BLOCK through data_block().
